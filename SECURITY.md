# Security policy

## Reporting a vulnerability

If you believe you've found a security issue in an Engine AI repo or service, do not open a public GitHub issue.

Email the founders directly:

- Ben du Chateau — benduchateau@gmail.com
- Joe Ward — (Joe's preferred contact)

Please include:

- A description of the issue
- Steps to reproduce
- Potential impact
- Any suggested mitigations

We'll acknowledge within 2 business days and keep you posted on the fix timeline.

## Secrets and credentials

If you accidentally commit a secret, credential, or API key to any Engine AI repo:

1. Notify a founder immediately. Do not wait.
2. Revoke the credential at the source.
3. We will force-expire the secret and rotate keys.
4. A decision log entry is required. Root cause gets captured.

## Responsible disclosure

We don't run a formal bug bounty. We do thank people who report issues responsibly, and we will credit reporters in release notes if they want.
